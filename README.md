# Spring Redis Coupon Service
Redis를 활용한 쿠폰 발급/관리 시스템 프로젝트

## 1. 프로젝트 배경

대규모 프로모션 이벤트(선착순 쿠폰 이벤트 등)에서는  
짧은 시간에 대량의 요청이 몰리며, 쿠폰 발급 로직이 서비스 전체 성능과 안정성에 직접적인 영향을 줍니다.

이전 실무 경험에서 특정 시간대에 API 호출이 집중되며, 동기 방식 처리로 인해 응답 지연이 발생하는 문제를 경험했습니다.  

이를 해결하기 위해 비동기 처리(@Async)를 도입하며, “**요청 수용과 실제 처리 로직을 분리하는 구조**”의 필요성을 체감했습니다.

이러한 경험을 바탕으로,  
**대량 트래픽 환경에서의 동시성 문제를 구조적으로 해결하는 방법**을 정리해보고자  
Redis 기반 선착순 쿠폰 발급 시스템을 설계·구현하였습니다.


## 2. 기존 RDB 기반 쿠폰 발급 구조의 한계

일반적인 RDB 기반 쿠폰 발급 구조는 쿠폰 수량 조회(SELECT) → 발급 처리(UPDATE) 방식으로 구성되는데,

이 구조는 동시 요청이 증가할수록, 트랜잭션 범위 확장과 Lock 경합으로 인해 TPS 저하와 응답 지연을 유발하게 되는 것으로 파악되었습니다.

이전 실무에서도 트랜잭션을 통해 데이터 정합성을 보장하려 할수록, 전체 처리 지연이 발생하는 상황을 경험했으며,  
쿼리 튜닝이나 인덱스 추가로 이슈 해결을 했던 경험이 있지만, 대규모 프로모션 이벤트 등의 경우에서는 선착순 요구사항을 안정적으로 만족시키기 어렵다는 한계를 느꼈습니다.
